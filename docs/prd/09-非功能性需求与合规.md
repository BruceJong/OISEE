# OISee PRD · 09 非功能性需求与合规

> 文档版本：V1.0
> 所属模块：全局 - 非功能性需求与合规
> 最后更新：2026-05-26

---

## 1. 概述

本文档描述 OISee 的非功能性需求（终端适配、性能、可用性等）与合规要求。这些要求约束所有功能模块，是产品质量与合法运营的基础。

> 注意：本文档只描述「需求与约束」，不涉及具体技术实现方案（实现方案见后续《技术设计文档》）。

---

## 2. 终端与适配

| 项目 | 要求 |
| --- | --- |
| 目标终端（当前阶段） | PC 网页 + 平板电脑 |
| 适配方式 | 响应式设计，自适应 PC 与平板 |
| 体验重点 | 平板是该年龄段孩子的高频终端，2.5D 探索体验在平板上需良好可用 |
| 远期规划 | 在网页产品打磨成熟后，再开发为 APP，以便快速迭代 |

> 当前阶段不要求适配手机端小屏，但响应式设计应预留向手机端延展的空间。

---

## 3. 性能与体验

| 项目 | 要求（方向性，非硬指标） |
| --- | --- |
| 加载体验 | 首页与场景探索需有良好的首屏加载体验；视频、2.5D 资源需考虑加载策略 |
| 流畅度 | 场景探索 / 2.5D 交互需保证在平板上的流畅度 |
| 媒体资源 | 视频、图片等大体积素材需有合理的加载 / 压缩策略（技术方案中细化） |

> 具体性能指标（如首屏时间阈值）待技术方案阶段结合实现确定。

---

## 4. 可用性与适龄设计

- 产品面向 6-16 岁青少年，UI / 交互 / 文案需**适龄、友好、易理解**。
- 低龄段（6-9 岁）用户识字与操作能力有限，关键引导需直观（图标 / 语音 / 动画等手段可在设计阶段考虑）。
- 安全相关提示（尤其动手实验）需清晰醒目（详见《06-动手实验》F-4）。

---

## 5. 合规要求（重点：未成年人保护）

OISee 全量用户为未成年人，合规是**硬约束**，开发前必须妥善处理。

### 5.1 未成年人个人信息保护

- 产品收集用户的用户名、年龄段、手机号等个人信息。
- 收集与处理未成年人个人信息须符合相关法律法规对**监护人同意**的要求。
- 注册环节（尤其手机号收集）需设置监护人同意机制。具体实现方式列为待确认事项（与《01-账号与权限》A-4 联动）。

### 5.2 内容合规

- 所有内容（含 AIGC 生成内容）面向青少年，需保证内容健康、科学准确、适龄。
- AIGC 生成内容的审核机制与《08-CMS》H-2 联动。

### 5.3 第三方登录合规

- 微信 / QQ 第三方登录需遵循对应平台规范及未成年人相关政策。

### 5.4 防沉迷 / 时长管理（待评估）

- 面向未成年人的产品通常需考虑使用时长管理 / 防沉迷机制。
- 当前需求未明确提出，列为待确认事项。

---

## 6. 数据与隐私

- 用户的探索度、积分、勋章、等级等成长数据需持久化存储（注册用户）。
- 匿名用户数据仅本地临时记录，不持久化（详见《01-账号与权限》）。
- 个人信息的存储、传输安全要求在技术方案中落实。
