## 3. 产品定位

| 项目 | 内容 |
| --- | --- |
| 产品名称 | OISee |
| 产品类型 | 面向青少年的科普网站 |
| 核心理念 | 通过探索身边的物品，了解其原理，从而建立对科学的认知，并养成底层思维逻辑 |
| 核心路径 | **场景 → 物品 → 知识点 → 实验** 的四级拆解链路 |
| 产品调性 | 以「引导孩子动手实验」为核心目的；场景探索环节适度游戏化，作为吸引孩子的「钩子」 |

### 3.1 核心理念阐述

OISee 不采用「先抛抽象知识，再要求记忆」的传统科普方式，而是反其道而行：

1. 从孩子熟悉的**生活场景**出发（如家、厨房）；
2. 在场景中发现具体的**物品**（如微波炉）；
3. 拆解物品背后的**科学知识点**（如电磁波加热原理）；
4. 通过**动手实验**把知识转化为亲身体验。

这条链路符合青少年「由具象到抽象」的认知规律，并通过游戏化机制持续激发探索欲望，最终把孩子引导到真实世界的动手实践中。

## 4. 目标用户

### 4.1 用户群体

- **主要使用者**：6-16 周岁青少年
- **关键相关方**：家长（陪伴者 / 购买决策者）

### 4.2 年龄分层

由于 6-16 岁认知跨度极大，产品将用户划分为三个年龄段，对应三级难度梯度。用户在注册时选择年龄段，确定其初始难度系数。

| 年龄段 | 难度等级 | 难度称号 | 认知特征 |
| --- | --- | --- | --- |
| 6-9 岁 | L1 初级 | 启蒙 | 关注现象与生活化类比（"是什么"） |
| 10-13 岁 | L2 中级 | 探索 | 关注原理与机制（"为什么"） |
| 14-16 岁 | L3 高级 | 深化 | 关注量化与学科化（"怎么算 / 如何关联"） |

> 难度体系的详细规则见《02-激励体系》。

## 5. 模块总览

OISee 分为**前台（用户端）**与**后台（内容管理端）**两大部分。

### 5.1 前台模块（用户端）

```
OISee 前台
├── 1. 首页（理念介绍 + 使用引导）
├── 2. 场景探索（游戏化地图）
│      └── 场景 → 物品列表 → 物品详情页
│                              ├── 原理介绍视频
│                              ├── Tab：物品爆炸图
│                              ├── Tab：知识点卡片
│                              └── Tab：一起来实验
├── 2a. 物品仓库（所有物品卡片库 + L1/状态筛选）
│      └── 物品详情页（与场景探索共用）
├── 3. 知识探索（知识点卡片库 + 知识网络）
│      └── 知识点详情 → 关联物品 + 关联实验
├── 4. 动手实验（实验卡片库）
│      └── 实验详情 → 演示视频 + 所需材料（材料包 / 家用物品）
└── 5. 我的书包（个人中心：探索度 / 等级 / 积分 / 勋章）
```

### 5.2 后台模块（内容管理端 CMS）

- 场景 - 物品 - 知识点 - 实验的关联关系管理（新增 / 修改 / 删除）
- 封面、视频、图片等素材上传
- 集成文生图 / 文生视频 AIGC Agent，辅助内容生产，并预置统一 prompt 保证风格一致

### 6.1 MVP 范围说明

MVP 阶段虽然内容上仅做「家-厨房」一个场景，但功能骨架是**完整**的：包含账号体系、CMS、AIGC 内容生产、完整激励体系。即第一版重点在于把整套「地基」立稳并打通一条端到端链路，后续版本主要是向其中填充更多场景与内容。

## 7. 术语表

| 术语 | 定义 |
| --- | --- |
| 场景（Scene） | 生活中的环境，如家、学校、医院、公园、商场等 |
| 物品（Item） | 场景中的具体对象，如微波炉、冰箱等 |
| 知识点（Knowledge Point） | 物品所承载的科学原理，带有学科与难度标签 |
| 实验（Experiment） | 与知识点关联的动手实践活动 |
| 难度系数 | 知识点与实验的分级标签（L1/L2/L3），对齐年龄段 |
| 探索度 | 用户对场景 / 物品 / 知识点的学习完成程度 |
| 知识网络 | 知识点之间相互关联形成的可视化图谱 |
| 材料包 | 实验所需材料的成品套装（二期可接入商城采购） |
| AIGC Agent | 后台集成的文生图 / 文生视频内容生产智能体 |
