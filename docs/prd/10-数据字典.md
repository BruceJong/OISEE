# OISee PRD · 10 数据字典

> 文档版本：V1.0
> 所属模块：全局 - 核心实体字段定义
> 最后更新：2026-05-26

---

## 1. 概述

本文档定义 OISee 核心业务实体的字段，作为前台功能、CMS 与后续技术设计的共同参照。本文档为**业务层字段定义**，不涉及数据库物理设计（表结构、索引、外键等在《技术设计文档》中定义）。

核心实体及关系：

```
场景（Scene） 1 ──< n 物品（Item） n >──< n 知识点（KnowledgePoint） n >──< n 实验（Experiment）
                                              │
                                              └──< n 知识点（知识网络，自关联）
用户（User） ──< 探索记录 / 积分 / 勋章 / 等级
```

> 多对多关系（如物品-知识点、知识点-实验、知识点-知识点）通过关联关系维护，由 CMS 配置。

---

## 2. 场景（Scene）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 场景 ID | 标识 | 是 | 唯一标识 |
| 名称 | 文本 | 是 | 如「家-厨房」「学校」 |
| 封面 | 图片 | 是 | 场景封面 / 地图图标 |
| 描述 | 文本 | 否 | 场景简介 |
| 是否默认开放 | 布尔 | 是 | 默认开放场景（如厨房）对匿名用户可见 |
| 解锁条件 | 规则 | 否 | 解锁该场景所需的完成度 / 条件（与激励体系联动，详见《02》《04》） |
| 地图位置 | 配置 | 否 | 在场景地图中的位置信息 |
| 物品列表 | 关联 | — | 包含的物品（一对多） |

---

## 3. 物品（Item）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 物品 ID | 标识 | 是 | 唯一标识 |
| 名称 | 文本 | 是 | 如「微波炉」 |
| 所属场景 | 关联 | 是 | 所属场景 |
| 封面 | 图片 | 是 | 物品封面 |
| 原理介绍视频 | 视频 | 否 | 物品详情页顶部视频（可能缺失，需兜底，详见《04》D-4） |
| 爆炸图 | 图片 | 是 | 拆解结构图（标注元件名称） |
| 元件标注 | 配置 | 否 | 爆炸图中各元件名称的标注信息 |
| 关联知识点 | 关联 | — | 该物品承载的知识点（多对多，含不同难度层） |
| 在场景中的位置 | 配置 | 否 | 场景 2.5D 图中的位置（若启用 2.5D） |

---

## 4. 知识点（KnowledgePoint）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 知识点 ID | 标识 | 是 | 唯一标识 |
| 名称 | 文本 | 是 | 知识点名称 |
| 学科 | 枚举 | 是 | 自然学科分类（枚举值见第 7 节，待最终确定） |
| 难度 | 枚举 | 是 | L1 初级 / L2 中级 / L3 高级 |
| 图文内容 | 富文本 | 是 | 知识点详细介绍（图文） |
| 卡片摘要 | 文本 | 否 | 卡片库展示用的简要信息 |
| 关联物品 | 关联 | — | 涉及该知识点的物品（多对多） |
| 关联知识点 | 关联 | — | 构成知识网络的其他知识点（多对多，自关联） |
| 关联实验 | 关联 | — | 对应的动手实验（多对多） |

> 难度标签决定该知识点对用户的开放 / 锁定状态（详见《02-激励体系》）。

---

## 5. 实验（Experiment）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 实验 ID | 标识 | 是 | 唯一标识 |
| 名称 | 文本 | 是 | 实验名称 |
| 难度 | 枚举 | 是 | L1 / L2 / L3，与关联知识点难度对齐 |
| 封面 | 图片 | 是 | 实验卡片封面 |
| 演示视频 | 视频 | 否 | 实验演示视频 |
| 所需材料-材料包 | 列表 | 否 | 材料包内含材料（二期可接入商城） |
| 所需材料-家用物品 | 列表 | 否 | 可用家中物品替代的材料 |
| 预计时长 | 数值 | 否 | 建议增强字段（详见《06》F-1） |
| 是否需家长陪同 | 布尔 | 否 | 建议增强字段 |
| 关联知识点 | 关联 | — | 对应的知识点（多对多） |
| 安全提示 | 文本 | 否 | 实验安全须知（详见《06》F-4） |

## 6. 用户（User）

> 用户相关字段与账号体系（《01》）、激励体系（《02》）联动。此处列业务层关键字段。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 用户 ID | 标识 | 是 | 唯一标识 |
| 用户名 | 文本 | 是 | 站内昵称 |
| 年龄段 | 枚举 | 是 | 6-9 / 10-13 / 14-16 岁，决定初始难度 |
| 当前难度系数 | 枚举 | 是 | L1 / L2 / L3，初始由年龄段决定 |
| 手机号 | 文本 | 否 | 手机号注册用户 |
| 第三方账号绑定 | 关联 | 否 | 微信 / QQ |
| 头像 | 图片 | 否 | 见《07》G-2 |
| 累计积分 | 数值 | 是 | 见《02》 |
| 当前等级 | 数值 | 是 | 由累计积分推导 |
| 已获勋章 | 关联 | — | 见《02》《07》 |

### 6.1 用户探索记录（派生数据）

| 数据 | 说明 |
| --- | --- |
| 已探索物品 | 用户探索过的物品集合 |
| 已学习知识点 | 用户学习过的知识点集合（点亮知识网络节点） |
| 已完成实验 | 用户完成的实验集合 |
| 场景探索度 | 各场景的完成程度（派生自知识点学习情况） |

## 7. 枚举值定义

### 7.1 年龄段

| 值 | 含义 |
| --- | --- |
| 6-9 岁 | 对应难度 L1 |
| 10-13 岁 | 对应难度 L2 |
| 14-16 岁 | 对应难度 L3 |

### 7.2 难度等级

| 值 | 称号 |
| --- | --- |
| L1 | 启蒙（初级） |
| L2 | 探索（中级） |
| L3 | 深化（高级） |

| 建议值 | 说明 |
| --- | --- |
| 物理 | |
| 化学 | |
| 生物 | |
| 地理 / 天文 | 待定是否拆分 |
| 其他 | 待补充 |

### 7.4 材料来源

| 值 | 含义 |
| --- | --- |
| 材料包 | 成品套装（二期可采购） |
| 家用物品 | 家中现有物品替代 |
