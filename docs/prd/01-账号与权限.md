# OISee PRD · 01 账号与权限

> 文档版本：V1.0
> 所属模块：前台 - 账号体系
> 最后更新：2026-05-26

---

## 1. 概述

OISee 采用「匿名可体验、注册得权限」的账号策略。匿名用户可体验默认场景以降低使用门槛；其余场景与个性化功能（探索度、积分、勋章等）需注册登录后使用，注册转化由「保留探索成果」作为钩子驱动。

用户在注册时选择年龄段，年龄段决定其初始难度系数，进而影响知识点的开放范围（详见《02-激励体系》）。

---

## 2. 访问权限分级

| 用户状态 | 可访问内容 | 限制 |
| --- | --- | --- |
| 匿名用户 | 仅默认场景（家-厨房）及其下属物品、知识点、实验；首页 | 不可访问其他场景；探索成果仅本地临时记录，不持久化 |
| 注册用户 | 全部场景（受解锁逻辑约束）、知识探索、动手实验、我的书包 | 受难度体系与解锁机制约束 |

### 2.1 匿名用户的探索成果处理

- 匿名用户在默认场景中产生的探索行为（如学习了哪些知识点）在**客户端本地临时记录**。
- 当匿名用户注册成功时，系统将本地临时记录的探索成果**迁移并合并**到新账号中。
- 该机制作为注册转化钩子，引导语示例：「注册即可保留你已探索的成果」。

---

## 3. 注册

### 3.1 注册方式

支持两种注册方式：

**方式一：手机号注册**

注册需填写以下信息：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 用户名 | 是 | 用户在站内的昵称 |
| 年龄段 | 是 | 三选一：6-9 岁 / 10-13 岁 / 14-16 岁，决定初始难度 |
| 手机号 | 是 | 用于验证与登录 |
| 短信验证码 | 是 | 手机号验证 |

**方式二：第三方登录注册**

- 支持微信、QQ 扫码注册。
- 第三方授权成功后，若为首次使用（无关联账号），需进入**信息补全**流程：
  - 补充**用户名**（必填）
  - 选择**年龄段**（必填）
- 补全完成后方可正式使用。

### 3.2 注册流程

```
[手机号注册]
输入用户名 + 年龄段 + 手机号
  → 获取并输入短信验证码
  → 验证通过
  → 注册成功（若由匿名转化，则合并本地探索成果）

[第三方注册]
选择微信 / QQ 扫码
  → 第三方授权
  → 判断是否首次使用
      ├── 是 → 补全用户名 + 年龄段 → 注册成功
      └── 否 → 直接登录
```

> 合规注意：涉及未成年人手机号等个人信息收集，须满足相关法规对监护人同意的要求，详见《09-非功能性需求与合规》。

---

## 4. 登录

- 已注册用户可通过手机号 + 验证码登录，或通过微信 / QQ 第三方登录。
- 第三方登录若识别为已有关联账号，直接登录，无需重复补全信息。

---

## 5. 年龄段与难度

- 年龄段在注册时确定，是用户难度系数的初始来源。
- 三个年龄段与三级难度的对应关系见《00-PRD-总览》及《02-激励体系》。
- 年龄段决定用户**默认可学习的知识点难度上限**；更高难度的知识点需通过解锁机制（勋章）开放。

> 关于「年龄段是否允许修改」「孩子升入更高年龄段后如何处理」等规则，列为**待确认事项**（见第 8 节）。

## 6. 账号与权限相关页面/状态清单

| 页面 / 状态 | 说明 |
| --- | --- |
| 注册页（手机号） | 用户名 + 年龄段 + 手机号 + 验证码 |
| 注册页（第三方补全） | 用户名 + 年龄段 |
| 登录页 | 手机号 / 第三方 |
| 受限提示 | 匿名用户访问非默认场景时的登录引导 |
| 成果迁移提示 | 匿名转注册时的成果保留引导 |

---

## 7. 边界与异常

- 用户名重复 / 不合规（违禁词）的校验规则待定义（见待确认事项）。
- 验证码超时、错误、重发的交互为通用规范，由设计阶段细化。
- 第三方授权失败时回退到登录/注册入口。

---

## 8. 待确认事项

| 编号 | 事项 | 说明 |
| --- | --- | --- |
| A-1 | 年龄段是否允许修改 | 例如孩子长大后跨入更高年龄段，是否支持调整及调整后探索成果如何处理 |
| A-2 | 家长账号是否需要 | 当前需求未提出家长独立账号 / 一号管多娃。是否在后续版本引入家长端，待定 |
| A-3 | 用户名校验规则 | 重复性、长度、违禁词过滤等 |
| A-4 | 未成年人监护人同意的具体实现方式 | 与合规要求联动，详见《09》 |
