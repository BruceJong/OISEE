## 1. 模块目标

物品仓库以**物品**为中心，把分散在所有二级场景下的物品**集中陈列**，作为「场景探索」之外的第二条物品访问路径。

> - **场景探索**：从场景（地图 → L1 建筑 → L2 子场景）出发寻找物品，强调"发现感"。
> - **物品仓库**：直接罗列所有物品，强调"全集 / 检索"，方便孩子对照自己已经探索过的物品和未探索的物品。
> - **知识探索**：从知识点出发，强调"网络化学习"。

三者并列于顶部导航：场景探索 · **物品仓库** · 知识探索 · 动手实验。

## 3. 一级页面：物品卡片库

### 3.1 内容

- **罗列数据库中所有 `PUBLISHED` 的物品**（不限场景），包括所有二级场景下挂载的物品。
- 每张卡片展示物品的核心信息：物品图、物品名、简介、所属一级场景 + 二级场景、知识点数、探索度。
- 卡片视觉与「知识探索」的 `KnowledgeCard` 保持一致（同款 `.card.lift` 外框、16:9 封面 + 左侧 4px 色条、底部"查看详情 →"行）。

### 3.2 筛选（仅 2 类）

| 筛选维度 | 类型 | 选项 |
| --- | --- | --- |
| **一级场景** | 多选 Checkbox | 我的家 / 公园 / 学校 / 医院 / 超市 / 商场 / 游乐场 |
| **探索状态** | 单选 Radio | 全部 / 未开始 / 进行中 / 已完成 |

- 取消了知识探索页中的"搜索/学科/排序"等其他筛选，回归极简："我想看 *什么场景* 的 *什么状态* 的物品"。
- 多选场景之间为「或」关系；场景与状态之间为「与」关系。
- 选中项在主区顶部显示为可点击的 tag chip，点 × 移除。

### 3.3 探索状态判定

物品探索度由 `utils/progress.ts` 计算（localStorage 持久化）：

```
itemProgress = 0.5 × KP 浏览比例 + 0.5 × 视频观看比例
（无视频时 = KP 浏览比例）
```

| 状态 | 判定 |
| --- | --- |
| 未开始 | `progress === 0` |
| 进行中 | `0 < progress < 0.99` |
| 已完成 | `progress >= 0.99` |

### 3.4 卡片布局规范

```
┌─────────────────────────────┐
│ ▌  [物品图 (Apple 产品摄影)]  ●N%│  ← 16:9 封面
│ ▌                            │     左 4px L1 主题色条
│ ▌  loc · 二级场景            │     右上探索度徽章
└─────────────────────────────┘
   [L1 tag]  [L2 tag]
   物品名
   简介（2-3 行）
   ─────── 进度条 ────────
   N KPs · 查看详情      →
```

- **封面**：物品图（Apple 产品摄影风格 1:1，居中 92%×92%），背景使用所属一级场景主题色 12% 浅色调。
- **左侧色条**：4px 宽，使用一级场景主题色。
- **右上徽章**：圆角胶囊，显示 `N%` 或 `✓ DONE`，已完成时反白。
- **左下场景路径**：mono 字体，`L1 · L2`，颜色用 L1 主题色。
- **正文**：tags（L1 + L2）→ 名称 → 简介 → 进度条 → "查看详情 →"。

### 3.5 物品图规范（图片素材生成）

| 项目 | 规范 |
| --- | --- |
| 风格 | Apple 产品摄影（clean studio, soft directional lighting, premium materials） |
| 背景 | 纯净浅灰渐变（隔离主体，便于嵌入卡片） |
| 构图 | 1:1 正方形，主体居中，下方有柔和阴影 |
| 尺寸 | 1024 × 1024，PNG |
| 命名 | `data/uploads/items/{item.slug}.png` |
| URL | `http://<host>/uploads/items/{slug}.png` 写入 `item.itemImageUrl` |
| 生成方式 | DashScope `qwen-image` 异步文生图 + Apple 风格提示词 |

### 3.6 统计信息

主区顶部显示 `{filtered}/{total} 个物品` 计数（mono 字体）。

## 5. 接口契约

### 5.1 `GET /api/v1/items`

| 项目 | 内容 |
| --- | --- |
| 权限 | 公开（前台） |
| 参数 | 无（MVP 不分页；后续若 >500 项可加 `cursor`） |

**返回**：

```ts
PublicItem[]
{
  id: string;
  slug: string;
  name: string;
  shortDesc: string;
  itemImageUrl: string | null;
  svgSymbolId: string | null;
  videoDurationSec: number | null;
  scene: {
    id: string;
    slug: string;       // L2 slug，如 'home-kitchen'
    name: string;       // L2 中文名，如 '厨房'
    groupName: string;  // L1 slug（'home'/'school'/...）
    iconKind: string | null;
    themeColor: string | null;
  };
  knowledgePoints: Array<{
    sortOrder: number;
    knowledgePoint: {
      id: string;
      slug: string;
      name: string;
      subject: 'PHYSICS' | 'CHEMISTRY' | 'BIOLOGY' | 'GEOGRAPHY' | 'OTHER';
      difficulty: 'L1' | 'L2' | 'L3';
    };
  }>;
}
```

- 查询条件：`status='PUBLISHED' && deletedAt is null`。
- 排序：`sortOrder asc, createdAt asc`。
- 物品的探索度**不在后端计算**，由前端从 localStorage 中按 `slug` 派生。

## 6. 内容管理（CMS）

物品仓库的内容来源完全沿用现有 `admin-items` 后台：
- 新增/编辑物品 → 立即出现在物品仓库。
- 物品图通过 CMS 上传或异步生成，写入 `itemImageUrl` 字段。
- `status` 从 DRAFT 切到 PUBLISHED 后立即在物品仓库露出。

---

## 7. 与场景探索的导航关系

- 卡片点击 → `/items/:slug` ItemDetailPage。
- ItemDetailPage 顶部"返回"按钮回到所属 L2 `/scenes/{l2 slug}`（沿用现有跳转），与物品仓库列表不强绑定。
- 顶部导航始终可点回 `/items` 重新筛选。

---

## 8. 非功能性

- 列表一次性返回所有 `PUBLISHED` 物品（MVP 阶段约 ≤500 项可接受），客户端做过滤、排序、分页。
- 卡片图通过 `<img>` 直接加载，未来若量大可上 lazy-loading 或虚拟列表。
- 状态/筛选切换不发请求，所有过滤在前端进行。
