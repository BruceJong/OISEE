> 文档版本：V2.0

## 1. 模块目标

CMS 是 OISee 的**内容生产中枢**，承担两个核心使命：

1. **运营基础设施**：所有前台内容（场景、物品、知识点、实验、关联关系、媒体素材）的录入与维护
2. **AI 内容工厂**：通过 LLM 与文生图/文生视频模型联动，把内容生产从"人工录入"升级为"人工编排 + AI 生成 + 人工审校"

> CMS 是 MVP 必须具备的"地基"之一：即使一期只上线"家-厨房"一个场景，也需要通过 CMS 配置该场景的全部内容。

## 2. 定位与边界

| 维度 | 用户端 | CMS |
| --- | --- | --- |
| 部署形态 | 公开 SPA | **独立 SPA**，与用户端分离 |

- MVP：主域共用，路径区分（`oisee.com/` 与 `oisee.com/admin/`）
- 二期可选：独立子域 `admin.oisee.com`，配合 IP 白名单

## 3. 账号与权限

### 3.1 账号策略

| 项 | 规则 |
| --- | --- |
| 注册入口 | **不提供任何公开注册** |
| 首次部署 | 系统 seed 一个超管账号（用户名/密码从部署配置读取） |
| 后续账号 | 仅"超管"或拥有"账号管理"权限的角色可添加 |
| 忘记密码 | **超管直接重置**：不走邮箱验证流程；重置后生成临时密码，首次登录强制改密 |
| 密码复杂度 | ≥ 10 位，含大小写、数字、符号 |
| 登录失败 | 每 IP 每分钟 5 次限流；连续 10 次失败锁定账号 30 分钟 |
| 敏感操作 | 查看完整手机号、删除内容、删除账号等需输入密码二次确认 |

### 3.2 权限模型（RBAC）

采用经典三层模型：**管理员 ↔ 角色 ↔ 权限节点**

- 一个管理员可分配多个角色（权限叠加）
- 一个角色拥有多个权限节点
- 权限节点分两类：
  - **菜单权限**：决定侧边导航中是否可见
  - **操作权限**：决定页面中具体按钮是否可点击（如"发布""删除""审核通过"）

### 3.3 MVP 角色配置

| 角色 | 范围 | 备注 |
| --- | --- | --- |
| 超级管理员 | 全部权限节点 | seed 时自动创建，唯一 MVP 内置角色 |

> RBAC 表结构完整保留，后续加角色（编辑/审核员/数据分析师等）通过 CMS 的"角色管理"页面动态添加，无需改代码。

## 5. 内容管理（核心模块）

### 5.1 通用能力

所有内容实体（场景/物品/知识点/实验）共享：

| 能力 | 说明 |
| --- | --- |
| 列表（分页、筛选、搜索、排序） | 支持按状态、关键词、创建时间筛选 |
| 详情编辑 | 表单 + 富文本 + 媒体上传 |
| 软删 | 删除后保留 30 天可恢复 |
| 发布态 | DRAFT → PUBLISHED → ARCHIVED |
| 操作审计 | 关键操作记录到审计日志 |

### 5.2 场景管理

#### 5.2.1 场景信息维护

字段对应《10-数据字典》中 `Scene` 实体：基础信息、分组、解锁规则、地图配置、视觉资产等。

新增配置项（V2 补充，反映原型设计）：

| 字段 | 说明 |
| --- | --- |
| 分组（group） | "家" / "校园" / "户外" / "城市" |
| 锁定提示文案 | 如"完成厨房 80% 探索度" |
| 地图图标类型 | 5 个预设：home / school / park / hospital / mall |
| 主色调 | 5 个预设：sun / ocean / leaf / coral / berry |

#### 5.2.2 场景 2.5D 大图

通过 AIGC 生成或手动上传，作为该场景的背景图。

#### 5.2.3 物品布局编辑器（场景管理的核心子能力）

**适用场景**：管理员需要为某场景的 2.5D 大图标定每个物品的位置时。

**交互流程**：

```
1. 进入场景详情 → "物品布局"标签页
2. 画布展示该场景的 2.5D 大图（按真实比例）
3. 右侧栏列出该场景下所有物品（已定位 / 未定位）
4. 拖动右侧物品到画布上 → 自动写入坐标
5. 已在画布上的物品可：
   ├── 拖拽改变位置
   ├── 拖角缩放改变大小
   └── 点击删除（仅清除坐标，物品本身不删）
6. 保存 → 用户端"场景探索"页面立即生效
```

> 这是 CMS 中**唯一一个所见即所得编辑器**。其本质是把 `Item.scenePosition` 的录入从"输入数字坐标"升级为"鼠标拖拽"。

#### 5.2.4 ✨ AI 智能联想：物品推荐

**触发**：场景创建后，详情页提供「智能推荐物品」按钮。

**业务逻辑**（混合模式）：

1. 系统输入：场景名 + 分组 + 描述
2. 系统输出：一组候选物品列表，每项标注"复用"或"新建"
3. 管理员行为：
   - 「复用」：直接关联现有物品到当前场景
   - 「新建」：跳转到物品新建页，名称/简介已预填

**示例**：
- 输入：场景"客厅"
- 输出：
  - 电视机（复用，已存在）
  - 空调（复用，已存在）
  - 沙发（新建，建议简介："家中最常坐的物品，藏着弹性原理"）
  - 加湿器（新建...）
  - ...

### 5.3 物品管理

#### 5.3.1 物品信息维护

字段对应《10-数据字典》中 `Item` 实体，V2 补充字段：

| 字段 | 说明 |
| --- | --- |
| 一句话简介（shortDesc） | 卡片悬停展示 |
| 一物三看（principleByLevel） | L1/L2/L3 三段原理摘要 |
| 零件清单（parts） | 爆炸图标注 `[{ no, name, desc, x, y }]` |
| 视频元信息 | 视频标题、时长 |

#### 5.3.2 ✨ AIGC 资产生成（4 种）

| 资产 | 用途 | 模态 |
| --- | --- | --- |
| 物品透明 PNG | 场景内贴图 | 文生图 |
| 物品爆炸图 | 物品详情页 Tab 1 | 文生图（特定 prompt） |
| 物品原理视频 | 物品详情页顶部 | 文生视频 |
| 物品封面 | 卡片缩略图 | 文生图 |

每种生成的交互：「生成」按钮 → 输入补充描述 → 异步任务，回填到对应字段。

#### 5.3.3 爆炸图标注编辑器

上传/生成爆炸图后，提供轻量标注工具：
- 在图上点击位置 → 添加标注点
- 输入零件名称与描述
- 标注点可拖动调整位置
- 写入 `Item.parts`

#### 5.3.4 ✨ AI 智能联想：知识点推荐

**触发**：物品详情页"关联知识点"区块的「智能推荐」按钮。

**业务逻辑**：与场景→物品推荐对称，输出按学科与难度分布建议的知识点列表，"复用"或"新建"标签呈现。

### 5.4 知识点管理

#### 5.4.1 知识点信息维护

| 区块 | 内容 |
| --- | --- |
| 基础信息 | 名称、slug、学科、难度、卡片摘要 |
| 图文内容 | 富文本（标题、段落、图片、视频嵌入、公式、提示框） |
| AI 辅助 | 「生成图文初稿」（基于名称 + 摘要）、「生成示意图」 |
| 关联关系 | 关联物品、关联实验、关联知识点（知识网络） |
| 测试题 | 测试题列表 + 「生成新测试题」 |

#### 5.4.2 知识网络关系编辑

**MVP 模式（列表式）**：

- 当前知识点为中心，展示已关联的知识点列表
- 「添加关联」按钮 → 弹窗：搜索 + 选择 + 标注关系类型（可选）
- 删除按钮

**增强模式（图谱式）**：

- 调用前台同款 3D 网络组件
- 可视化拖拽建立关联
- MVP 不做，二期考虑

#### 5.4.3 ✨ 知识点测试题生成

**业务定位**：

- CMS 阶段就具备测试题生产能力
- 前台 MVP 不渲染入口（PRD 已明确测验是二期前台功能）
- 数据建好后，二期前台**零成本**接入

**生成交互**：

```
知识点详情页 → 「测试题」Tab → 「生成新测试题」按钮 → 弹窗：
  - 题型：单选 / 多选 / 判断
  - 难度：简单 / 中等 / 难
  - 数量：1-5 题
  - 补充说明：可选自定义提示
点击生成 → AI 调用 → 返回题目列表（草稿态）→ 管理员逐题审校 → 发布
```

**题目数据**：题干、选项、正确答案、解析。

### 5.5 实验管理

#### 5.5.1 实验信息维护

字段对应《10-数据字典》中 `Experiment` 实体。

#### 5.5.2 ✨ AI 生成实验方案

**触发**：实验编辑页 → 选定 1-2 个关联知识点 → 「根据知识点生成实验方案」按钮。

**生成内容**：

| 字段 | 来源 |
| --- | --- |
| 名称 | AI 生成 |
| 简介 | AI 生成（30-50 字，激发好奇心） |
| 预计时长 | AI 生成 |
| 是否需要家长陪同 | AI 判断 |
| 安全提示 | AI 生成 |
| 材料清单（kit + household） | AI 生成 |
| 实验步骤 | AI 生成（5-8 步） |
| 现象与原理解释 | AI 生成 |

**管理员行为**：

- 所有字段**可全部编辑覆盖**
- 材料清单提供表格增删改
- 步骤用富文本编辑
- 生成后可点「生成演示视频」继续 AIGC

### 6.1 任务列表

实时展示所有 AIGC 任务，支持按状态、类型、关联实体筛选。

字段：任务 ID、类型、关联实体、状态、创建人、创建时间、完成时间、耗时、操作（查看 / 取消 / 重试 / 预览结果）。

### 6.2 Prompt 模板管理

| 类型 | 用途 |
| --- | --- |

## 7. AI 提供商配置（系统设置）

### 7.1 核心诉求

支持**多种 LLM、文生图、文生视频模型按 OpenAI 接口调用规范接入**，用户在 CMS 设置页面配置模型与 Key，无需改代码。

### 7.2 业务规则

| 规则 | 说明 |
| --- | --- |
| 三模态隔离 | LLM / IMAGE / VIDEO 独立维护提供商列表 |
| 单激活策略 | 同模态同时只一个激活的提供商；切换需手动激活 |
| 激活前必须测试 | 强制连通性测试通过才允许激活，避免业务中断 |
| 任务绑定 | 任务创建时记录当时激活的 provider；切换不影响在途任务 |
| 密钥安全 | 加密存储；查询接口仅显示末 4 位 |

### 7.3 系统预设（seed）

部署时 seed 主流服务商的空配置（仅有 name/baseUrl/defaultModel，密钥待填）：

测试结果（通过/失败 + 延迟 + 错误信息）展示在卡片上。

## 8. C 端用户管理

### 8.1 业务范围

| 能力 | 是否支持 |
| --- | --- |
| 查看用户列表 | ✅ |
| 查看用户详情（含探索度/积分/勋章） | ✅ |
| 查看完整手机号（脱敏切换） | ✅（需二次确认 + 审计） |
| 禁用 / 启用账号 | ✅（需填写理由 + 审计） |
| 修改用户积分 / 等级 / 勋章 | ❌（不在 MVP，避免越权后台发奖） |
| 删除用户 | ❌（注销走用户自助流程，二期） |

### 8.2 用户列表

字段：用户名、手机号（脱敏）、年龄段/难度、累计积分/等级、勋章数、注册时间、最后登录、状态、操作。

筛选：用户名、手机号（精确）、状态、年龄段、注册时间范围。

### 8.3 禁用规则

- 禁用即写入用户状态为 DISABLED
- 下次该用户的 token 校验时直接拒绝
- 操作进审计日志，需填写理由
- 禁用不删除数据，可启用恢复

## 9. 监控面板（仪表盘）

### 9.1 数据卡片

四张实时数据卡片：

| 指标 | 计算口径 |
| --- | --- |
| 总用户数 | 全表用户数 |
| 今日活跃 | 当日有登录的用户数 |
| 今日注册 | 当日新注册用户数 |
| 今日积分 | 当日发放积分总和 |

每张卡片附带与前一日的对比涨跌。

### 9.2 趋势图表

- **用户增长趋势**：近 30 天每日新注册用户折线图
- **活跃热度**：各场景的学习+实验完成数柱状图
- **最热知识点 Top 10**：学习次数倒序列表

### 9.3 内容统计

- 各类内容数量（场景/物品/知识点/实验）
- 已发布率
- 待审核数量

### 9.4 AIGC 任务状态

- 待执行任务数
- 执行中任务数
- 今日完成数
- 今日失败数

### 9.5 时段切换

数据卡片与折线图支持「今日 / 近 7 天 / 近 30 天」切换。

### 9.6 实时性要求

- AIGC 任务状态：5 秒刷新（接近实时）
- 数据卡片：30 秒刷新

> 不强求 WebSocket/SSE，轮询足够。

## 12. 验收清单（MVP）

CMS 完成 MVP 的判定标准：

- [ ] 超管账号可登录，进入仪表盘
- [ ] 场景：可新建、编辑 2.5D 大图、拖拽布局物品、发布
- [ ] 物品：可新建、4 种 AIGC 生成、爆炸图标注、关联知识点
- [ ] 知识点：可新建、富文本编辑、配图生成、知识网络关联、测试题生成（草稿态）
- [ ] 实验：可基于知识点生成方案、编辑材料、生成演示视频
- [ ] AI 智能联想：场景→物品、物品→知识点 端到端可用
- [ ] AIGC 工作台：任务可视、可重试、可取消
- [ ] Prompt 模板：可在 CMS 管理
- [ ] AI 提供商：seed 主流服务商空配置；可填 key、测试、激活；切换不影响在途任务
- [ ] C 端用户：可查看、可禁用，操作进审计
- [ ] RBAC：角色/权限表已建，超管账号默认拥有全部权限节点
- [ ] 忘记密码：超管可直接重置其他管理员密码，重置后首次登录强制改密
- [ ] 仪表盘：数据卡片 + 图表 + AIGC 状态实时展示
- [ ] 操作审计：所有关键操作有日志，可查询
