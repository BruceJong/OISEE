# OISee PRD · 05 前台功能 - 知识探索

> 文档版本：V1.0
> 所属模块：前台 - 知识探索
> 版本归属：MVP（一期）
> 最后更新：2026-05-26

---

## 1. 模块目标

知识探索模块以「知识点」为中心组织内容，提供知识点的**罗列、筛选、搜索**，并通过**知识网络**将知识点相互关联，强化孩子的记忆点，形成结构化的知识认知。

> 与场景探索的区别：场景探索是「从物品发现知识」的探索路径；知识探索是「以知识点为索引」的检索与网络化学习路径。两者互为补充。

---

## 2. 页面层级

```
知识点卡片库（一级）
  ├── 筛选 / 搜索
  ├── 知识网络（可视化图谱）
  └── 知识点详情页（二级）
        ├── 知识点详细介绍
        ├── 关联物品
        └── 关联实验
```

---

## 3. 一级：知识点卡片库

### 3.1 内容

- 罗列所有知识点卡片。
- 每张卡片展示知识点的核心信息（名称、学科、难度等，具体字段见《10-数据字典》）。

### 3.2 筛选

支持以下筛选维度，且可**综合筛选**（多维度组合）：

| 筛选维度 | 说明 |
| --- | --- |
| 自然学科 | 按学科分类筛选（如物理、化学、生物等，分类标准见《10-数据字典》） |
| 难度系数 | 按 L1 / L2 / L3 筛选 |

### 3.3 搜索

- 支持直接搜索对应知识点（按名称 / 关键词）。

### 3.4 难度与权限

- 知识点卡片库中，高于用户当前难度的知识点同样遵循**可见但锁定**的规则（与物品详情页内的知识点卡片保持一致，详见《02-激励体系》《04-场景探索》）。

---

## 4. 知识网络

### 4.1 定义

- 每个知识点关联若干**其他知识点**以及相关**物品**，从而形成一个知识点网络。
- 知识网络用于强化记忆、揭示知识点之间的内在联系，是产品的差异化亮点。

### 4.2 呈现形式（建议）

- 建议以**可视化图谱**（节点 + 连线）形式呈现知识网络，节点为知识点，连线表示关联关系。
- 可视化图谱具有较强的记忆点与「探索发现」的体验价值。

### 4.3 关联关系来源

- 知识点之间、知识点与物品之间的关联关系，由内容团队通过 CMS 配置维护（详见《08-内容管理后台CMS》）。

## 5. 二级：知识点详情页

### 5.1 整页布局（V2，与物品详情对齐）

```
┌─────────────────────────────────────────────────────────────┐
│ ← 返回 {来源}                                                │
│                                                              │
│  [难度] [学科]                                                │
│  知识点名（H1）                                               │
│  一句简介（lead）                                              │
│                                                              │
│  ┌──────────────────────────┬──────────────────────────┐   │
│  │                          │ ／ USED IN · 关联物品  全部→│   │
│  │   原理介绍视频卡          │  ◫ 物品 A                 │   │
│  │   （16:9，带 ▶ 播放按钮）  │  ◫ 物品 B                 │   │
│  │                          │  ◫ 物品 C                 │   │
│  │                          │  ◫ 物品 D                 │   │
│  │                          │                            │   │
│  │                          │ ／ RELATED · 关联知识点 全部→│  │
│  │                          │  ◇ 知识点 A               │   │
│  │                          │  ◇ 知识点 B  ...           │   │
│  └──────────────────────────┴──────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  [原理详解]  [考考你 N]                                       │
│  ───────────────────                                         │
│                                                              │
│  Tab 内容（按选中切换）                                       │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 顶部区块（视频 + 侧栏）

- **左：视频卡**（16:9）—— 知识点的原理介绍短视频
  - 数据字段：`videoTitle` / `videoDurationSec` / `videoUrl`
  - 若该 KP 难度高于当前账号难度 → 显示"🔒 等级不足"，播放按钮禁用
  - 视频未配置时显示 illustrationUrl 作为占位海报

- **右：侧栏**（与物品详情相同布局节奏）
  - 关联物品：最多展示 4 条；超出显示 `全部 N 条 →`，点击弹框列全部
  - 关联知识点：最多展示 4 条；超出同样可"全部"
  - 卡片样式与列表页一致（带物品图 / 学科色条）

### 5.3 底部 Tabs

| Tab | 内容 |
| --- | --- |
| **原理详解** | 图文 + 示意图（CMS 配置），左主区右摘要卡 |
| **考考你** | 单选题列表，回答后即时显示正误 + 解析（详见 5.4） |

二期再添加：动手实验 / 动画演示等 Tab。

### 5.4 「考考你」交互规则

- 每个 KP 至少 3 道 4 选 1 单选题（冷启动）
- 答题流程：
  1. 选中某个选项（仅一次可改）
  2. 点击「提交答案」
  3. 提交后显示：正确选项绿色 ✓ / 错误选项红色 ✗ / 解析文案
- 顶部统计：`已答 X/N · 答对 Y`
- 答题状态在客户端 localStorage 中保存（仅本会话有效，不入服务端进度）
- 题型来源：`quizQuestions[]` 数组（数据模型见 5.6）

### 5.5 跳转关系

| 触发点 | 跳转目标 |
| --- | --- |
| 知识点卡片 | 知识点详情页 |
| 知识网络节点 | 对应知识点详情页 |
| 侧栏 / 弹框 - 关联物品 | 物品详情页（《04》） |
| 侧栏 / 弹框 - 关联知识点 | 另一个知识点详情页（递归） |
| 顶部"返回" | 通过 `backStack` 栈回到来源（物品 / 知识库 / 上一层 KP），栈空时回 `/knowledge` |

### 5.6 数据模型变更

`KnowledgePoint` 新增字段：
- `videoTitle: String?` — 视频标题
- `videoDurationSec: Int?` — 时长（秒）
- `videoUrl: String?` — 视频地址（OSS / CDN）

新增 `QuizQuestion` 实体（详见《10-数据字典》）：
- `knowledgePointId` FK → KnowledgePoint
- `question` 题干
- `choices: string[]` 4 个选项
- `correctIndex: int` 正确答案下标
- `explanation: string?` 解析
- `difficulty: L1|L2|L3`
- `sortOrder` / `status`

## 6. 学习与积分

- 用户「看完」一个知识点（图文 / 视频）即视为学习该知识点，触发积分（详见《02-激励体系》）。
- 学习知识点会点亮知识网络中的对应节点，并计入学科类勋章进度。
- 「考考你」答题积分计算规则在《02》中追加（一道全对 = +N 分）；本期仅前端展示，不计入积分。
