# OISee PRD · 05 前台功能 - 知识探索

> 文档版本：V1.0
> 所属模块：前台 - 知识探索
> 版本归属：MVP（一期）
> 最后更新：2026-05-26

---

## 1. 模块目标

知识探索模块以「知识点」为中心组织内容，提供知识点的**罗列、筛选、搜索**，并通过**知识网络**将知识点相互关联，强化孩子的记忆点，形成结构化的知识认知。

> 与场景探索的区别：场景探索是「从物品发现知识」的探索路径；知识探索是「以知识点为索引」的检索与网络化学习路径。两者互为补充。

---

## 2. 页面层级

```
知识点卡片库（一级）
  ├── 筛选 / 搜索
  ├── 知识网络（可视化图谱）
  └── 知识点详情页（二级）
        ├── 知识点详细介绍
        ├── 关联物品
        └── 关联实验
```

---

## 3. 一级：知识点卡片库

### 3.1 内容

- 罗列所有知识点卡片。
- 每张卡片展示知识点的核心信息（名称、学科、难度等，具体字段见《10-数据字典》）。

### 3.2 筛选

支持以下筛选维度，且可**综合筛选**（多维度组合）：

| 筛选维度 | 说明 |
| --- | --- |
| 自然学科 | 按学科分类筛选（如物理、化学、生物等，分类标准见《10-数据字典》） |
| 难度系数 | 按 L1 / L2 / L3 筛选 |

### 3.3 搜索

- 支持直接搜索对应知识点（按名称 / 关键词）。

### 3.4 难度与权限

- 知识点卡片库中，高于用户当前难度的知识点同样遵循**可见但锁定**的规则（与物品详情页内的知识点卡片保持一致，详见《02-激励体系》《04-场景探索》）。

---

## 4. 知识网络

### 4.1 定义

- 每个知识点关联若干**其他知识点**以及相关**物品**，从而形成一个知识点网络。
- 知识网络用于强化记忆、揭示知识点之间的内在联系，是产品的差异化亮点。

### 4.2 呈现形式（建议）

- 建议以**可视化图谱**（节点 + 连线）形式呈现知识网络，节点为知识点，连线表示关联关系。
- 可视化图谱具有较强的记忆点与「探索发现」的体验价值。

### 4.3 关联关系来源

- 知识点之间、知识点与物品之间的关联关系，由内容团队通过 CMS 配置维护（详见《08-内容管理后台CMS》）。

---

## 5. 二级：知识点详情页

### 5.1 内容

| 区块 | 内容 |
| --- | --- |
| 知识点详细介绍 | 该知识点的详细原理讲解（图文） |
| 关联物品 | 该知识点所涉及的物品，可跳转至对应物品详情 |
| 关联实验 | 与该知识点关联的动手实验，可跳转至对应实验详情 |

### 5.2 跳转关系

| 触发点 | 跳转目标 |
| --- | --- |
| 知识点卡片 | 知识点详情页 |
| 知识网络节点 | 对应知识点详情页 |
| 详情页 - 关联物品 | 物品详情页（《04》） |
| 详情页 - 关联实验 | 实验详情页（《06》） |

---

## 6. 学习与积分

- 用户「看完」一个知识点（图文 / 视频）即视为学习该知识点，触发积分（详见《02-激励体系》）。
- 学习知识点会点亮知识网络中的对应节点，并计入学科类勋章进度。
- 知识点小测验为**二期功能**，本期不做。

## 7. 待确认事项

| 编号 | 事项 | 说明 |
| --- | --- | --- |
| E-1 | 自然学科分类标准 | 学科枚举值需在《10-数据字典》中最终确定，并与学科类勋章一致 |
| E-2 | 知识网络的展示规模与性能 | 知识点数量增长后图谱的可读性与性能策略 |
| E-3 | 「看完即学会」的判定标准 | 如视频是否需播放完、图文是否需停留等，设计阶段细化 |
| E-4 | 知识网络是否区分难度层 | 锁定态知识点在网络中如何呈现（灰显 / 隐藏） |
