## 1. 概述

本文档定义 OISee 的数据库 Schema、实体关系、索引策略与典型查询场景。物化为 Prisma Schema（`code/apps/api/prisma/schema.prisma`），由 Prisma 生成 TypeScript 类型供共享层使用。

设计原则：
- **多对多用显式关联表**：避免 Prisma 隐式关联表难以加扩展字段
- **业务标识与主键分离**：所有实体使用 `cuid()` 主键，业务可读 ID（slug）作为唯一索引
- **删除全部走软删除**：内容运营高频改动，需保留历史
- **难度/学科等枚举入数据库**：便于将来运营自助扩展，但 MVP 用 Prisma enum 即可


```prisma
model Scene {
  id            String        @id @default(cuid())
  slug          String        @unique                  // 业务标识，如 home-kitchen
  name          String                                  // 厨房
  groupName     String                                  // 场景分组："家" / "校园" / "户外" / "城市"

  coverUrl      String                                  // 封面图 OSS URL
  sceneImageUrl String?                                 // 2.5D 大图 OSS URL（缺失时前端用 SVG 模板）
  iconKind      String?                                 // 地图节点 SVG 模板：home/school/park/hospital/mall

  isDefault     Boolean       @default(false)           // 是否默认开放给匿名用户
  unlockRule    Json?                                   // 解锁规则（结构化 JSON）

  progresses    SceneProgress[]
  aiTasks       AiTask[]                                // 该场景关联的 AIGC 任务

  deletedAt     DateTime?

  @@index([status, sortOrder])
}

model Item {
  id                  String   @id @default(cuid())
  slug                String   @unique
  name                String
  sceneId             String
  scene               Scene    @relation(fields: [sceneId], references: [id])

  coverUrl            String
  itemImageUrl        String?                            // 透明背景物品图（覆盖在场景 2.5D 上）
  svgSymbolId         String?                            // 缺位图时使用的内置 SVG 模板 id（如 'microwave'）

  // 原理视频
  principleVideoUrl   String?                            // 原理介绍视频
  videoTitle          String?                            // 视频标题
  videoDurationSec    Int?                               // 视频时长（秒）
```

## 4. 设计要点说明

### 4.1 软删除与发布态

- 所有内容实体（Scene/Item/KP/Experiment）有：
  - `status: DRAFT | PUBLISHED | ARCHIVED`（业务发布态）
  - `deletedAt: DateTime?`（物理软删，仅管理员可见已删数据）
- 前台只读接口默认过滤 `status = PUBLISHED AND deletedAt IS NULL`。

### 4.2 `slug` 字段

- 每个内容实体都有 `slug`（如 `home-kitchen`、`microwave-oven`），用于：
  - URL 友好（`/scenes/home-kitchen`）
  - 数据导入/导出时人类可读
  - 跨环境内容同步
- 由 CMS 在创建时自动生成（拼音），管理员可修改。

### 4.3 难度与解锁规则的存储

PRD 中的解锁规则示例：
- 解锁某物品的 L2 知识点：集齐该物品全部 L1 知识点 + 拥有 ≥3 枚任意勋章
- 解锁某物品的 L3 知识点：集齐该物品全部 L2 知识点 + 拥有 ≥1 枚实验类勋章

MVP 实现策略：
- 规则**硬编码**在规则引擎中（详见 [05](./05-激励规则引擎.md)），不存数据库
- 用户解锁的具体知识点写入 `UserUnlock`
- 后期需要"运营自助配置规则"时，再考虑把规则结构化存表

### 4.4 知识网络的有向 vs 无向

`KnowledgeRelation` 是有向边（`fromId → toId`），但**业务上视为无向**——查询时合并两侧：

```sql
SELECT * FROM knowledge_relation
WHERE from_id = $1 OR to_id = $1
```

如果需要严格无向（避免重复），CMS 录入时强制 `fromId < toId`，前端图谱构建按无向处理。

### 4.5 积分流水（`PointsLedger`）

- 每笔积分变动**单独入账**，不直接 `UPDATE user.totalPoints`
- `User.totalPoints` 是缓存值，写积分时事务内同步更新
- `idempotencyKey` 保证幂等：例如 `LEARN_KP:user_abc:kp_xyz` 同一用户同一知识点只能加一次分

### 4.6 等级派生

- `User.level` 不入流水，由 `totalPoints` 派生
- 等级阈值表硬编码在规则引擎，写积分后同步重算
- 当等级提升时，可触发"升级"勋章（如有）

### 4.7 AIGC 任务关联多实体

`AiTask` 表用 4 个可空外键（`sceneId / itemId / knowledgePointId / experimentId`）关联到对应实体。这种"宽表"在 Prisma 不算优雅，但比"多态关联"清晰。每个任务只关联一个实体，由 `type` 决定哪个字段有值，应用层在创建任务时做约束。

## 5. 索引策略

| 表 | 索引 | 用途 |
| --- | --- | --- |
| `Scene` | `(status, sortOrder)` | 前台列表 |
| `Item` | `(sceneId, sortOrder)` | 场景内物品列表 |
| `KnowledgePoint` | `(subject, difficulty)` | 知识探索筛选 |
| `KnowledgePoint` | `(status)` | 前台过滤已发布 |
| `Experiment` | `(difficulty, status)` | 实验筛选 |
| `User` | `(phone)` | 登录 |
| `PointsLedger` | `(userId, createdAt)` | 个人积分流水时间倒序 |
| `AiTask` | `(status, createdAt)` | 队列消费 |
| `AiTask` | `(thirdPartyTaskId)` | DashScope 回调匹配 |

## 9. 与 PRD 数据字典的差异说明

PRD 数据字典是"业务字段视角"，本文档是"数据库视角"，存在以下增补/差异：

| 增补字段 | 原因 |
| --- | --- |
| `slug` | URL 友好与跨环境同步 |
| `status / deletedAt` | 发布态与软删 |
| `sortOrder` | 列表展示顺序 |
| `idempotencyKey` (PointsLedger) | 幂等保证 |
| `AuditLog` | 管理员审计（合规） |
| `RefreshToken` | JWT 刷新机制 |
| `Scene.groupName` | 场景分组（家/校园/户外/城市），驱动地图分区与勋章计数 |
| `Scene.iconKind / themeColor` | 地图节点 SVG 模板与配色，与原型呼应 |
| `Scene.unlockHint` | 锁定提示文案，提升锁定态体验 |
| `Item.shortDesc` | 卡片悬停一句话简介 |
| `Item.principleByLevel` | "一物三看"区块的三层原理摘要 |
| `Item.parts` | 爆炸图零件清单（取代原 `explodedAnnotations`，结构更丰富） |
| `Item.svgSymbolId` | 缺位图时使用的内置 SVG 模板，承担兜底 |
| `Item.videoTitle / videoDurationSec` | 视频元数据，前端展示用 |
| `AIProvider` | AI 服务商配置（多 Provider 抽象，由 CMS 管理） |
| `AiTask.providerId` | 绑定任务执行时的 Provider 快照 |

| 暂未建模字段 | 原因 |
| --- | --- |
| 实验"安全提示"的结构化展示 | MVP 用单字段 `safetyNotes` 文本，需要分级再拆 |
| 用户家长账号 | PRD A-2 待确认事项，不建模 |
| 防沉迷时长统计 | PRD I-2 待确认，预留 |
