## 1. 概述

本文档定义 OISee 代码仓库的物理组织方式、依赖管理、构建链路、开发环境与代码规范，是后续所有开发工作的工程基线。

设计目标：
- **单人开发友好**：少配置、低心智负担、AI 生成代码风格稳定
- **类型贯通**：后端 Prisma 类型 → 共享层 → 前端，零手工同步
- **本地与生产一致**：Docker Compose 一份编排两套环境

## 2. 仓库布局（Monorepo）

采用 **pnpm workspace + Turbo（可选）** 的 Monorepo。结构：

```
oisee/
├── apps/
│   ├── web/                    # 用户端前端
│   │   ├── src/
│   │   ├── public/
│   │   ├── index.html
│   │   ├── vite.config.ts
│   │   ├── tsconfig.json
│   │   └── package.json
│   ├── admin/                  # CMS 前端
│   │   ├── src/
│   │   ├── ...（结构同 web）
│   │   └── package.json
│   └── api/                    # NestJS 后端
│       ├── src/
│       │   ├── modules/
│       │   ├── common/
│       │   ├── main.ts
│       │   └── app.module.ts
│       ├── prisma/
│       │   ├── schema.prisma
│       │   ├── migrations/
│       │   └── seed.ts
│       ├── test/
│       ├── nest-cli.json
│       ├── tsconfig.json
│       └── package.json
│
├── packages/
│   ├── shared/                 # 共享类型 / 常量 / 枚举 / Zod schema
│   │   ├── src/
│   │   │   ├── types/          # 从 Prisma 转出的业务类型
│   │   │   ├── schemas/        # Zod 校验 schema（前后端共用）
│   │   │   ├── constants/      # 难度等级、积分常量、错误码
│   │   │   └── index.ts
│   │   ├── tsconfig.json
│   │   └── package.json
│   └── config/                 # 共享的 eslint / prettier / tsconfig
│       ├── eslint.config.js
│       ├── prettier.config.js
│       ├── tsconfig.base.json
│       └── package.json
│
├── docker/
│   ├── api.Dockerfile
│   ├── web.Dockerfile
│   ├── admin.Dockerfile
│   ├── nginx.conf
│   └── nginx.Dockerfile
│
├── scripts/
│   ├── seed-dev.ts             # 本地开发种子数据
│   ├── backup-db.sh            # 数据库备份脚本
│   └── deploy.sh               # 部署脚本
│
├── docker-compose.yml          # 生产编排
├── docker-compose.dev.yml      # 本地开发（只起 PG + Redis）
├── pnpm-workspace.yaml
├── package.json                # root：scripts、devDeps
├── turbo.json                  # （可选）任务编排
├── .env.example
├── .gitignore
└── README.md
```

### 2.1 为什么这样切

- `apps/` 是可独立部署的应用，`packages/` 是仅供内部依赖的库。
- `web` 与 `admin` 分两个应用而非一个 SPA：①CMS 的 AntD 体积大、加载策略不同；②可以独立部署、独立鉴权域；③用户端打开速度不受 CMS 影响。
- `shared` 是关键：所有跨应用类型/schema 都从这里走，避免接口对接错位。
- `config` 集中放共享工具配置，避免每个 app 各自维护。

## 3. 包依赖关系

```
        ┌──────────┐
        │  shared  │ ◄────── code/apps/web
        └────┬─────┘ ◄────── code/apps/cms
             │       ◄────── code/apps/api
             ▼
        （Prisma 生成类型 + Zod schema）

        ┌──────────┐
        │  config  │ ◄────── 所有 apps & packages 引用 eslint/tsconfig
        └──────────┘
```

> **重要规则**：`shared` 不依赖任何 app，apps 之间互不依赖。所有"前后端共享"的内容（类型、错误码、常量、Zod schema）都进 `shared`。

## 4. pnpm Workspace 配置

### 4.1 `pnpm-workspace.yaml`

```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

### 4.2 根 `package.json` 关键脚本

  "name": "oisee",

    "db:generate": "pnpm --filter @oisee/api prisma generate",

### 4.3 子包命名

| 路径 | 包名 |
| --- | --- |
| `code/apps/web` | `@oisee/web` |
| `code/apps/cms` | `@oisee/cms` |

## 5. TypeScript 配置策略

### 5.1 `code/packages/config/tsconfig.base.json`（共享基础）

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "forceConsistentCasingInFileNames": true,
    "noUncheckedIndexedAccess": true,
    "noImplicitOverride": true
  }
}
```

### 5.2 各应用 `tsconfig.json` 继承基础并覆盖

- `code/apps/web` / `code/apps/cms`：`"jsx": "react-jsx"`，`"lib": ["DOM", "ES2022"]`
- `code/apps/api`：`"module": "CommonJS"`（NestJS 默认 CJS），`"experimentalDecorators": true`，`"emitDecoratorMetadata": true`
- `code/packages/shared`：构建为 dual ESM/CJS（用 `tsup`），导出 `.d.ts`

### 5.3 路径别名

各 app 内统一使用 `@/*` 指向 `src/*`，避免相对路径 `../../../`。共享包用包名导入：`import { ItemSchema } from '@oisee/shared'`。

## 6. 共享层 `@oisee/shared` 设计

### 6.1 内容范围

| 子目录 | 内容 |
| --- | --- |
| `src/types/` | Prisma 生成的模型类型 re-export + 业务派生类型（如 `ItemWithKnowledgePoints`） |
| `src/schemas/` | Zod schema：DTO、查询参数、表单校验 |
| `src/constants/` | 枚举常量：难度等级、年龄段、学科、错误码 |
| `src/utils/` | 纯函数工具：难度比较、积分计算辅助等 |

### 6.2 关键示例

```ts
// code/packages/shared/src/constants/difficulty.ts
export const DIFFICULTY = {
  L1: 'L1',
  L2: 'L2',
  L3: 'L3',
} as const;
export type Difficulty = typeof DIFFICULTY[keyof typeof DIFFICULTY];

export const AGE_BAND = {
  AGE_6_9: '6-9',
  AGE_10_13: '10-13',
  AGE_14_16: '14-16',
} as const;
export type AgeBand = typeof AGE_BAND[keyof typeof AGE_BAND];

export const AGE_BAND_TO_DIFFICULTY: Record<AgeBand, Difficulty> = {
  [AGE_BAND.AGE_6_9]: DIFFICULTY.L1,
  [AGE_BAND.AGE_10_13]: DIFFICULTY.L2,
  [AGE_BAND.AGE_14_16]: DIFFICULTY.L3,
};
```

```ts
// code/packages/shared/src/schemas/auth.ts
import { z } from 'zod';

export const PhoneRegisterSchema = z.object({
  username: z.string().min(2).max(20),
  ageBand: z.enum(['6-9', '10-13', '14-16']),
  phone: z.string().regex(/^1[3-9]\d{9}$/),
  smsCode: z.string().length(6),
});

export type PhoneRegisterInput = z.infer<typeof PhoneRegisterSchema>;
```

> 后端 NestJS 用 Zod 做 DTO 校验（通过 `nestjs-zod`），前端 React Hook Form 用同一 schema 校验。**前后端校验逻辑零分裂**。

## 7. 代码规范与质量

### 7.1 ESLint（共享配置）

`code/packages/config/eslint.config.js` 导出 flat config，各 app `eslint.config.js` 引用：

```js
import shared from '@oisee/config/eslint.config.js';
export default [...shared, { /* app 特定规则 */ }];
```

核心规则：
- `@typescript-eslint/strict`
- `eslint-plugin-import`（强制按层级排序）
- `eslint-plugin-unicorn`（精选规则）
- 禁用：`no-explicit-any`（强制类型）、`no-floating-promises`（防 Promise 未 await）

### 7.2 Prettier

统一格式化：单引号、行宽 100、尾逗号 all。

### 7.3 Git Hooks

- **husky + lint-staged**：提交前对暂存文件跑 prettier + eslint --fix
- **commitlint**：约束 commit message 为 `<type>(<scope>): <subject>` 格式
- 提交类型：`feat | fix | refactor | docs | chore | test | perf`

### 7.4 编辑器

仓库根目录提供 `.vscode/` 推荐配置：
- 推荐插件：Prisma、ESLint、Prettier、Tailwind IntelliSense
- workspace 设置：保存时格式化、自动 organize imports

## 8. 本地开发环境

### 8.1 前置依赖

| 工具 | 版本 |
| --- | --- |
| Node.js | 22 LTS |
| pnpm | 9.x |
| Docker Desktop | 最新 |
| PostgreSQL 客户端（可选） | 16 |

### 8.2 `docker-compose.dev.yml`（只起依赖服务）

```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: oisee
      POSTGRES_PASSWORD: oisee_dev
      POSTGRES_DB: oisee_dev
    ports:
      - '5432:5432'
    volumes:
      - oisee_pg_dev:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    ports:
      - '6379:6379'
    volumes:
      - oisee_redis_dev:/data

volumes:
  oisee_pg_dev:
  oisee_redis_dev:
```

### 8.3 启动流程

```bash
# 一次性
pnpm install
pnpm docker:dev              # 起 PG + Redis
cp .env.example .env         # 填本地配置
pnpm db:migrate              # 应用 Prisma 迁移
pnpm db:seed                 # 灌入种子数据（管理员账号 + 厨房场景）

# 日常
pnpm dev                     # 并行启动 api + web + admin
```

启动后端口分配：

| 服务 | 端口 | URL |
| --- | --- | --- |
| api (NestJS) | 3000 | http://localhost:3000 |
| web (Vite) | 5173 | http://localhost:5173 |
| admin (Vite) | 5174 | http://localhost:5174 |
| postgres | 5432 | |
| redis | 6379 | |

前端通过 Vite proxy 把 `/api/*` 反代到 `http://localhost:3000`，避免 CORS。

## 10. 构建与产物

| 应用 | 构建命令 | 产物 |
| --- | --- | --- |
| `@oisee/web` | `vite build` | `code/apps/web/dist/` 静态资源 |
| `@oisee/cms` | `vite build` | `code/apps/cms/dist/` 静态资源 |
| `@oisee/api` | `nest build` | `code/apps/api/dist/` Node.js 可执行 |
| `@oisee/shared` | `tsup src/index.ts --format esm,cjs --dts` | `code/packages/shared/dist/` |

构建顺序：`shared` → 各 app（Turbo 自动按依赖排序）。

## 11. Turbo（可选但推荐）

`turbo.json` 配置增量构建与缓存：

```json
{
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**"]
    },
    "dev": { "cache": false, "persistent": true },
    "lint": {},
    "typecheck": {}
  }
}
```

好处：仅改 `web` 的代码时，`shared` 不会重复构建；`pnpm typecheck` 也能并行。

> 单人项目可不上 Turbo，pnpm `-r --parallel` 已够用。规模上来后再引入。

## 12. 数据库种子数据策略

`code/apps/api/prisma/seed.ts` 在本地开发与首次部署时灌入：

1. **管理员账号**：1 个超级管理员（账号/密码从 `.env` 读取）
2. **基础枚举**：学科枚举如不入表则跳过，入表则写入
3. **MVP 内容**（仅本地）：厨房场景 + 微波炉物品 + 3 个示例知识点 + 2 个示例实验

生产环境只跑 `seed:prod`（只灌管理员账号），不灌示例内容。
