## 1. 概述

本文档定义用户端（`code/apps/web`）和管理端（`code/apps/cms`）两个前端应用的架构。两者使用相同的技术基础（React + Vite + TypeScript + Zustand + TanStack Query），但 UI 体系和组件库不同：

| 维度 | 用户端 (web) | 管理端 (admin) |
| --- | --- | --- |
| 目标用户 | 6-16 岁青少年 + 家长 | 内容运营 / 管理员 |
| UI 库 | **无 UI 库**，自研组件 + CSS Modules + CSS Variables | Ant Design + AntD Pro 组件 |
| 设计风格 | **编辑刊物 + 蓝图科技感**（沿用原型设计系统） | 严谨、信息密集、高效 |
| 终端 | PC + 平板（响应式） | PC 为主 |
| 加载策略 | 关键页骨架屏 + 资源预加载 | 按需懒加载 |

> **设计风格说明**：用户端遵循原型的"克制专业风"——大留白、细线条、Noto Serif SC 衬线标题、Space Grotesk mono 标签、深靛蓝 + 琥珀单色强调。详见 [08-原型分析](./08-原型分析与技术调整.md) §2。

### 2.3 状态管理策略

| 数据类型 | 工具 | 说明 |
| --- | --- | --- |
| 服务器数据（场景/物品/知识点等） | TanStack Query | 自动缓存、并发去重、失效控制 |
| 用户认证状态（user, token） | Zustand + localStorage 持久化 | 跨标签同步 |
| UI 状态（Tab、Modal） | useState / Zustand | 全局用 Zustand，局部 useState |
| 匿名浏览记录 | Zustand + localStorage 持久化 | 注册后清空 |
| 全局反馈（Toast / 勋章弹窗队列） | Zustand | 多个勋章串行展示 |

### 2.4 API 客户端

`api/client.ts`：

```ts
const client = axios.create({
  baseURL: '/api/v1',
  timeout: 15000,
});

// 请求拦截：附加 token
client.interceptors.request.use((config) => {
  const token = useAuthStore.getState().accessToken;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// 响应拦截：解包 + token 过期自动刷新
client.interceptors.response.use(
  (res) => {
    const { code, data, message } = res.data;
    if (code === 0) return data;
    throw new BusinessError(code, message);
  },
  async (err) => {
    if (err.response?.status === 401 && /* 未重试 */) {
      await useAuthStore.getState().refresh();
      return client(err.config);
    }
    throw err;
  }
);
```

### 2.5 2.5D 场景渲染方案（SVG/位图双模式）

> 设计原则：开发期与生产期一致的坐标系统与交互层，资产可以是矢量 SVG 模板或 CMS 上传的位图。详见 [08-原型分析](./08-原型分析与技术调整.md) §4.2。

#### 2.5.1 资产抽象

```ts
type SceneAsset =
  | { kind: 'svg'; component: ComponentType }                      // 内置 SVG 模板
  | { kind: 'image'; url: string; viewBox: { w: number; h: number } };  // CMS 上传位图

type ItemAsset =
  | { kind: 'svg'; symbolId: string }                              // 复用 SVG 物品组件
  | { kind: 'image'; url: string };                                // 透明 PNG/WebP

interface SceneItemDescriptor {
  id: string;
  slug: string;
  name: string;
  shortDesc: string;
  knowledgePointNames: string[];        // 悬停气泡显示前 3 个
  asset: ItemAsset;
  position: { x: number; y: number; width: number; height: number };  // 百分比 0-100
}
```

#### 2.5.2 数据来源映射

后端返回的物品布局：
```json
{
  "sceneImageUrl": "https://cdn/.../kitchen.png",      // 缺失时前端走 SVG 模板
  "iconKind": "home",                                  // 兜底 SVG 模板 id
  "items": [
    {
      "slug": "microwave",
      "name": "微波炉",
      "shortDesc": "发出看不见的「波」让食物变热",
      "itemImageUrl": "https://cdn/.../microwave.png", // 缺失时走 svgSymbolId
      "svgSymbolId": "microwave",
      "scenePosition": { "x": 30, "y": 38, "width": 12, "height": 14 },
      "knowledgePoints": [
        { "name": "电磁波是什么" },
        { "name": "水分子的振动" },
        { "name": "介电加热（L3）" }
      ]
    }
  ]
}
```

`position` 采用**百分比坐标**（0-100），与场景图自身的纵横比无关，前端按容器宽度自动缩放。

#### 2.5.3 渲染实现

```tsx
// SceneRenderer.tsx
function SceneRenderer({ sceneAsset, items }: SceneRendererProps) {
  const [hoveredId, setHoveredId] = useState<string | null>(null);

  return (
    <div className={styles.sceneContainer}>
      {sceneAsset.kind === 'svg'
        ? <sceneAsset.component />
        : (
          <img
            src={sceneAsset.url}
            className={styles.sceneBg}
            style={{ aspectRatio: `${sceneAsset.viewBox.w}/${sceneAsset.viewBox.h}` }}
          />
        )}

      {items.map((item) => (
        <ItemHotspot
          key={item.id}
          item={item}
          hovered={hoveredId === item.id}
          onHover={setHoveredId}
        />
      ))}

      {/* 悬停气泡：当前 hovered 物品的标题 + 知识点 */}
      {hoveredId && <HoverBubble item={items.find(i => i.id === hoveredId)!} />}
    </div>
  );
}
```

```tsx
// ItemHotspot.tsx
function ItemHotspot({ item, hovered, onHover }) {
  const navigate = useNavigate();
  return (
    <button
      className={cx(styles.hotspot, hovered && styles.hovered)}
      style={{
        left:   `${item.position.x}%`,
        top:    `${item.position.y}%`,
        width:  `${item.position.width}%`,
        height: `${item.position.height}%`,
      }}
      onMouseEnter={() => onHover(item.id)}
      onMouseLeave={() => onHover(null)}
      onClick={() => navigate(`/items/${item.slug}`)}
      aria-label={item.name}
    >
      {item.asset.kind === 'svg'
        ? <SvgSymbol id={item.asset.symbolId} />
        : <img src={item.asset.url} alt={item.name} />}
    </button>
  );
}
```

#### 2.5.4 资产回退策略

```
sceneImageUrl 存在？─Y→ 用位图
              └─N→ 用内置 SVG 模板（按 iconKind）

itemImageUrl 存在？──Y→ 用位图
             └─N→ 用内置 SVG 模板（按 svgSymbolId）
             └─都缺失 → 用 emoji + 名字的兜底图
```

#### 2.5.5 CMS 配置工作流

1. CMS 新建场景 → 先用 SVG 模板占位（管理员可立即看到效果）
2. 管理员调用 AIGC 生成 2.5D 大图，或手动上传 → `sceneImageUrl` 填充
3. 管理员在大图预览上拖拽物品贴图 → 写入 `scenePosition`（百分比）
4. AIGC 生成或上传物品透明 PNG → `itemImageUrl` 填充

#### 2.5.6 性能优化

- 位图走 OSS 图片处理：`?x-oss-process=image/resize,w_1920/format,webp`
- 首屏场景图 `<img fetchpriority="high">` 预加载
- SVG 模板组件按需 `React.lazy` 加载（仅当场景需要时）
- 物品悬停气泡用 CSS 动画，不触发 React 重渲染
- 平板：`touch-action: manipulation` 消除 300ms 点击延迟

### 2.6 知识网络可视化（自研 3D 物理仿真）

> 直接迁移自 `front/OSEE原型/src/Knowledge.jsx` 的 `NetworkView` 实现。详见 [08-原型分析](./08-原型分析与技术调整.md) §4.3。

#### 2.6.1 视觉与交互

- **节点**：知识点以彩色圆形呈现，颜色按学科映射（物理蓝/化学红/生物绿/地理紫），锁定态灰显
- **连线**：知识点间的关联关系（来自 `KnowledgeRelation`），淡白色细线，高亮时变琥珀色
- **3D 透视**：节点 z 坐标决定近大远小与堆叠顺序，远处节点会被前景遮挡
- **物理仿真**：
  - 节点间斥力（避免堆叠）
  - 关联边弹性（保持目标距离）
  - 中心引力 + 阻尼（系统稳定）
- **交互**：
  - 拖拽空白处：旋转视角（Y/X 双轴）
  - 单击节点：选中 + 高亮关联线 + 弹出名称气泡
  - 双击节点：跳转知识点详情
  - 空闲 2 秒后：自动绕 Y 轴缓慢旋转

#### 2.6.2 实现要点

```tsx
function KnowledgeNetwork({ nodes, edges, userLevel, onOpenDetail }) {
  const [selected, setSelected] = useState<string | null>(null);
  const stateRef = useRef<SimulationState>();        // 节点位置 + 旋转角度
  const containerRef = useRef<HTMLDivElement>(null);

  // 初始化：按学科分布在球面上不同区域
  if (!stateRef.current) {
    stateRef.current = initSimulation(nodes);
  }

  // 物理仿真循环
  useAnimationFrame(() => {
    applyRepulsion(stateRef.current.nodes);
    applySpring(stateRef.current.nodes, edges);
    applyCenterGravity(stateRef.current.nodes);
    integrate(stateRef.current.nodes);
    if (stateRef.current.auto) {
      stateRef.current.rotation.ry += 0.0035;
    }
  });

  // 拖拽旋转（mousedown/move/up + 触摸事件）
  useDragRotation(containerRef, stateRef);

  // 透视投影
  const projected = projectAll(stateRef.current);

  return (
    <div ref={containerRef} className={styles.canvas}>
      <Stars />                                              {/* 星空背景 */}
      <NetworkEdges projected={projected} edges={edges} selected={selected} />
      <NetworkNodes
        projected={projected}
        nodes={nodes}
        selected={selected}
        userLevel={userLevel}
        onSelect={setSelected}
        onOpenDetail={onOpenDetail}
      />
      <NetworkLegend />
      <NetworkHint />
    </div>
  );
}
```

#### 2.6.3 性能与降级

- 数据量 < 100 节点：主线程跑物理仿真，60fps 无压力
- 数据量 100-300：把仿真挪到 **Web Worker**，主线程只做渲染
- 检测到低端设备（`navigator.hardwareConcurrency < 4`）或用户偏好（`prefers-reduced-motion`）：降级为 2D 静态布局（按学科分簇）

#### 2.6.4 关联场景

知识网络的视觉语言与「我的书包 - 知识星图」共享：
- 同样的星空背景（径向暗蓝渐变）
- 同样的脉冲与高亮规则
- 已学习的知识点为发光节点，未学习为暗淡节点

### 2.7 关键交互反馈

#### 2.7.1 积分飘字

学习/完成实验后，调用 `feedbackStore.showPointsToast(+10)`：屏幕右上角飘出 +10 动画。

#### 2.7.2 勋章解锁弹窗

后端返回 `newBadges` 时，加入 Zustand 队列，串行弹窗（避免同时多个）：

```ts
const newBadges = response.newBadges;
newBadges.forEach((b) => feedbackStore.enqueueBadge(b));
```

#### 2.7.3 升级弹窗

`levelUp: true` 时，先排队升级弹窗，再排队勋章弹窗。

### 2.8 适龄设计要点

- **大按钮**（最小点击区 48x48px）
- **强对比色**，避免低龄看不清
- **避免长文本**，每段控制在 2-3 行
- **进度可视化**为主（条、星、地图点亮），数字为辅
- **声音反馈**（可选）：点击物品、获得勋章时有提示音
- **响应式断点**：
  - `< 768px`（手机，二期）
  - `768px - 1024px`（平板竖屏）
  - `1024px - 1440px`（平板横屏 / 小笔记本）
  - `> 1440px`（PC）

### 3.3 CMS 关键交互

#### 3.3.1 物品在场景中的位置编辑器（`ItemPositionEditor`）

```
┌─────────────────────────────────────────┐
│ 场景 2.5D 大图（缩略图，1:1 比例容器）    │
│                                           │
│   ┌─────┐                                 │
│   │微波 │  ← 拖拽 / 缩放                    │
│   │ 炉  │                                 │
│   └─────┘                                 │
│                                           │
│            ┌──┐                           │
│            │冰│                           │
│            │箱│                           │
│            └──┘                           │
└─────────────────────────────────────────┘
[ 物品列表，点击高亮其位置 ]
```

实现：使用 [react-rnd](https://github.com/bokuweb/react-rnd) 或自己用拖拽 + ResizeObserver 实现。

#### 3.3.2 爆炸图标注（`ExplodedAnnotator`）

在爆炸图上**点击**添加标注点，输入元件名称：
```
{ "x": 120, "y": 80, "label": "磁控管" }
```
存到 `Item.explodedAnnotations`。

前端展示时，绝对定位标签气泡。

#### 3.3.3 知识网络关系编辑（`RelationsEditor`）

提供两种编辑模式：
1. **列表模式**：当前知识点 + 已关联知识点列表，按钮"添加关联"打开搜索弹窗
2. **图谱模式**（增强）：React Flow 显示局部网络，可视化拖拽建立连线

MVP 用列表模式即可。

#### 3.3.4 AIGC 生成按钮（`AigcGeneratorButton`）

在编辑表单中复用，比如物品编辑页有三个按钮：
- 生成原理视频
- 生成爆炸图
- 生成物品贴图

点击按钮 → 弹窗输入补充描述 → 调用 `POST /admin/aigc/tasks` → 关闭弹窗 → 启动轮询（每 5s 查任务状态）→ 完成后自动回填到表单字段。

### 3.4 富文本编辑（知识点内容）

使用 **TipTap**：
- 扩展：图片、视频嵌入、代码块、提示框、数学公式（KaTeX）
- 图片/视频通过 OSS 直传，插入 URL
- 保存为 TipTap JSON，存入 `KnowledgePoint.content`
- 前台用 `@tiptap/react` 渲染模式（只读 + 自定义样式）

### 3.5 表单与校验

统一封装 `ZodForm` 组件，对接 React Hook Form + `@hookform/resolvers/zod`：

### 3.6 OSS 直传上传组件

```tsx
function ImageUpload({ purpose, onChange }) {
  const upload = async (file: File) => {
    const sign = await api.media.signUpload({ fileName: file.name, fileType: file.type, purpose });
    await axios.put(sign.uploadUrl, file, { headers: sign.headers });
    await api.media.notify({ objectKey: sign.objectKey });
    onChange(sign.publicUrl);
  };
  // ...
}
```

## 4. 共用前端能力

### 4.1 视频播放

- 选用 **xgplayer**（西瓜视频，对阿里云 VOD 支持好）
- 支持多码率自适应（HLS）
- 关键事件：`onPlay`、`onEnded`、`onProgress`
- 后台可配置：是否必须看完才能"学会"（PRD E-3 待确认）

### 4.2 学习行为埋点

简化版 analytics（MVP 不接第三方）：
- 内置事件：`page_view`、`item_explored`、`kp_learned`、`experiment_completed`
- 仅用于内部计数（写后端 `progress`），不发到第三方

### 4.3 错误兜底

- React Error Boundary 包在每个 Page 外层
- 网络错误：Toast 提示 + 重试按钮
- 资源加载失败（图/视频）：占位图 + "刷新重试"按钮
- 视频缺失（PRD D-4）：使用预置插画 + 文字"视频即将上线"

### 4.4 SEO（用户端）

- 用户端是 SPA，对 SEO 不强求
- 但首页可通过 `react-helmet-async` 设置 meta
- 后期若需 SEO，可在 Nginx 层做 Prerender 或转 Next.js（架构预留可能）
