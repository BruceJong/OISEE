## 1. 原型概述

`front/OSEE原型/` 是一份**单页可运行的 React 原型**：
- 9 个完整页面（首页 / 场景地图 / 厨房场景 / 物品详情 / 知识库 / 知识详情 / 实验库 / 实验详情 / 我的书包）
- 完整的设计系统（颜色、字体、字号、组件库）
- 大量**自绘 SVG 矢量资产**（厨房 2.5D、爆炸图、物品图标、地图、Hero 插画、知识封面）
- **自研 3D 物理仿真知识网络**（拖拽旋转 + 自动旋转 + 物理弹簧 + 透视投影）
- 完整的 Mock 数据（`data.js`）
- 加载方式：浏览器 + Babel Standalone（无构建），单文件 HTML 即可访问

技术栈：React 18 + JSX(Babel) + 纯 CSS Variables + 内联 SVG。**未使用任何 UI 库或 CSS 框架**。

## 2. 设计风格归纳

| 维度 | 原型设定 |
| --- | --- |
| 风格定位 | **编辑刊物 + 蓝图科技感**（"Editorial × Blueprint"） |
| 设计原则 | 大留白 / 细线条 / 自绘 SVG / 安静配色 |
| 中文正文字体 | Noto Sans SC |
| 中文标题字体（衬线） | Noto Serif SC（用于 h1/h2/h3） |
| 英文/数字字体 | Space Grotesk（用作 `font-mono`，mono 标签风格） |
| 主色 | 纸感米白 `#F8F6F1` + 深靛蓝 `#0E1A33` + 琥珀 `#D89531`（唯一强调色） |
| 学科配色 | 物理蓝 / 化学红 / 生物绿 / 地理紫 |
| 难度配色 | L1 绿 / L2 琥珀 / L3 紫（与学科色 token 分离，避免耦合） |
| 装饰元素 | 「DWG·001·MICROWAVE」「SCALE 1:1·PRINCIPLE FIRST」等蓝图角标、网格背景 |
| 圆角 | 极克制（多数 0 圆角，仅卡片 4-8px） |
| 阴影 | 几乎不用，靠 1px hairline 分隔 |

> **关键判断**：这不是「儿童化彩色卡通」风格，而是**面向 6-16 岁的"克制专业风"**——把孩子当读者，而不是当被哄的对象。这个调性需要在技术选型中保留。

## 4. 调整项详述

### 4.1 用户端 UI 体系——放弃 AntD，沿用原型设计系统

#### 决策

| 应用 | UI 方案 |
| --- | --- |
| 用户端 `code/apps/web` | **CSS Modules + CSS Variables（沿用原型 tokens）+ 极少量公共组件**，不引入 AntD/Tailwind |
| CMS 端 `code/apps/cms` | **继续 Ant Design + Ant Design Pro 组件**（工具风，与原型艺术风格无关，毋庸更换） |

#### 理由

- 原型的视觉强度**远超 AntD 默认风格**，强行用 AntD 会需要大量覆盖样式，得不偿失
- Tailwind 在大量"非系统化"自定义样式（如内联 style、SVG 配色）下会显得多余
- CSS Variables + CSS Modules 的组合：
  - 维护成本低（设计 token 集中）
  - AI 生成代码可读性高（不需要记忆原子类）
  - 与原型代码迁移路径最短

#### 落地

- `code/apps/web/src/styles/tokens.css` —— 直接迁移原型 `styles.css` 的 `:root` 变量
- `code/apps/web/src/styles/global.css` —— 基础 reset + 字体加载
- 组件样式用 `Component.module.css`
- 仍可装 Tailwind 仅作为辅助布局工具（如 grid/flex 快写），但**禁止使用其颜色 token**，颜色一律走 CSS Variables

#### 影响范围

- `04-前端架构设计.md` §1（技术栈表）需更新 UI 库选型
- `04-前端架构设计.md` §3.5（CMS 富文本/表单）保持不变

### 4.2 2.5D 场景——支持「SVG 矢量」+「位图」双模式

#### 决策

抽象统一的 `<SceneRenderer />`，支持两种资产形态：

```tsx
type SceneAsset =
  | { kind: 'svg'; component: ReactComponentType }       // 自绘 SVG（开发期）
  | { kind: 'image'; url: string; viewBox: [w, h] };     // CMS 上传位图（生产期）

type ItemAsset =
  | { kind: 'svg'; symbolId: string }                    // 复用 SVG.Appliance 组件库
  | { kind: 'image'; url: string };                      // 透明 PNG

interface SceneItem {
  id: string;
  asset: ItemAsset;
  position: { x: number; y: number; width: number; height: number };  // 百分比 0-100
  label: string;
  knowledgeBrief: string[];                              // 悬停气泡显示的 3 行知识点
  onClick: () => void;
}
```

#### 渲染策略

```
                ┌────────────────────────────────────┐
                │  <SceneRenderer/>                  │
                │                                    │
                │  if asset.kind === 'svg':          │
                │    <BackgroundComponent />         │
                │  else:                             │
                │    <img src={asset.url} />         │
                │                                    │
                │  items.map(item =>                 │
                │    <Hotspot                        │
                │      position={item.position}      │
                │      asset={item.asset}            │
                │      ...                           │
                │    />                              │
                │  )                                 │
                └────────────────────────────────────┘
```

#### 理由

- 用户已明确："场景、封面等在原型中是占位图，后期从 CMS 获取" → 生产路径必须是 CMS 上传的位图
- 但原型证明了 SVG 矢量方案有独特优势：
  - 文件极小（一个厨房场景 SVG 可能只有 10-20KB）
  - 无损缩放，平板高分辨率下完美锐利
  - 可参数化配色（如锁定态可以一键变灰）
- 因此：**保留 SVG 模式作为开发期/兜底方案**，CMS 上传位图作为正式发布资产
- 当 CMS 未配置位图时，渲染器自动回退到 SVG 模板

#### CMS 工作流

```
1. 管理员新建场景"客厅"
2. CMS 默认使用 SVG 占位（提示"此场景使用临时素材"）
3. 管理员上传 2.5D 大图 → SVG 模板被替换
4. 管理员在大图上**用鼠标拖拽定位**每个物品 → 写入 position
5. 管理员上传每个物品的透明 PNG → ItemAsset 替换
```

#### 影响范围

- `04-前端架构设计.md` §2.5（2.5D 渲染方案）需重写
- `02-数据模型设计.md` `Scene` 与 `Item` 字段保持兼容，但语义变化
- 新增 `code/apps/web/src/scene-svg/` 目录存放自绘 SVG 资产（厨房、客厅、浴室、学校等）

### 4.3 知识网络——改为自研 3D 物理仿真

#### 决策

**放弃 React Flow，复用原型的 `NetworkView` 实现**。

#### 关键代码（已在原型中实现）

- 物理引擎（节点斥力 + 边弹性 + 中心引力 + 阻尼）
- 3D 透视投影（Y-X 双轴旋转矩阵）
- 鼠标拖拽旋转 + 空闲自动旋转
- 节点按 z 距离排序与缩放
- 选中节点 + 高亮关联线 + 弹出名称气泡
- 学科配色 + 锁定态灰显
- 双击进详情、单击选中

#### 理由

- 原型方案的视觉冲击和"宇宙感"远超 React Flow，**这是产品差异化亮点**
- 自研代码已经在原型中可运行，迁移成本低
- React Flow 优势是节点丰富的交互（连接、编辑），用户端只是浏览，用不上
- "我的书包"页面也有"知识星图"概念，可与知识网络共享视觉语言

#### 性能考量

| 节点数 | 表现 | 优化 |
| --- | --- | --- |
| < 100 | 流畅 | 无需优化 |
| 100-300 | 主线程可能压力 | 把物理仿真挪到 Web Worker |
| > 300 | 必须分层显示 | 按学科聚类，未选中学科节点折叠为聚合点 |

> MVP 仅 1 个场景，知识点估算 < 50 个，性能完全无忧。

#### 影响范围

- `04-前端架构设计.md` §2.6（知识网络）需重写
- 删除 React Flow 依赖

### 4.5 路由策略——保留 React Router，但对齐原型 IA

| 原型 route | 生产 URL | 备注 |
| --- | --- | --- |
| `home` | `/` | |
| `scenes` | `/scenes` | 全屏地图 |
| `scenes` + modal | `/scenes?open=home-kitchen` | 点击场景弹窗用 query string |
| `kitchen-scene:home-kitchen` | `/scenes/home-kitchen` | 场景内部页（2.5D） |

### 4.6 自绘 SVG 资产层——长期保留

原型的 SVG 不只是"占位图"，其中有些是**长期保留的设计资产**：

| 资产 | 性质 | 处理 |
| --- | --- | --- |
| 场景 2.5D 大图（厨房、客厅...） | 占位，CMS 上传位图替换 | `code/apps/web/src/scene-svg/`，作为 fallback |
| 物品图标（微波炉、冰箱...） | 占位，CMS 上传透明 PNG 替换 | `code/apps/web/src/item-svg/`，作为 fallback |

| 知识点封面图 | 占位，按学科自动生成 | `code/apps/web/src/kp-cover/` 按 subject 派生 |
| Hero 插画 | **永久保留**，开发期固定资产 | `code/apps/web/src/assets/hero/` |
| 探索地图（含河流、道路、树）| **永久保留**，开发期固定资产 | `code/apps/web/src/assets/map/` |
| 装饰图标（蓝图角标、网格背景） | **永久保留** | `code/apps/web/src/assets/decor/` |
| 各种小图标（实验玻璃瓶等） | **永久保留** | `code/apps/web/src/assets/icons/` |
| 实验完成 Confetti、知识星图 | **永久保留**，反馈组件 | `code/apps/web/src/components/feedback/` |

#### 关键原则

**业务内容资产（场景图/物品图/视频）** 由 CMS 提供，可被 AIGC 生成或人工上传；**设计资产（Hero/地图底图/装饰元素）** 是工程代码的一部分，跟着代码迭代。

### 4.7 字体加载策略

| 字体 | 原型来源 | 生产策略 |
| --- | --- | --- |
| Noto Sans SC | Google Fonts | 自托管至 OSS，提供 woff2 |
| Noto Serif SC | Google Fonts | 同上 |
| Space Grotesk | Google Fonts | 同上 |

#### 实施

1. 从 [Google Fonts Helper](https://gwfh.mranftl.com/fonts) 或 npm 包 `@fontsource/*` 下载 woff2
2. 中文字体按需子集化（用 `subfont` 工具扫描文案，仅生成包含实际使用字符的子集，可从 ~10MB 降到 ~500KB）
3. CDN 域名 `cdn.oisee.com` 提供字体
4. 用 `<link rel="preload">` 预加载关键字体
5. `font-display: swap` 避免 FOIT

| 字体子集化漏字 | 部分汉字显示宋体回退 | 子集字符集包含 GB2312 + 高频生僻字；落地后做一次全文扫描 |
| CSS Modules + 大量内联样式混用 | 风格分裂 | 制定规范：仅一次性动态值（如百分比定位）用内联，其他全走 Modules |
| 原型代码迁移引入运行时 bug | 上线延期 | 关键组件（知识网络、SceneRenderer）补单测；首屏可视化回归脚本 |
