# OISee 部署实操指南（当前仓库结构）

> 最后更新：2026-06-17
> 适用：单台阿里云 ECS + Docker Compose，**本机磁盘存储**（OSS 后续接入）
> 关联：[07-部署与运维方案](./07-部署与运维方案.md)（架构/备份/监控理念，但其中的 Dockerfile 路径已过时，以本篇为准）

---

## 0. 这套部署文件包含什么

仓库根目录已生成（按当前 `backend/ + frontend-user/ + frontend-admin/ + shared/lib/` 结构写好）：

| 文件 | 作用 |
| --- | --- |
| `docker/api.Dockerfile` | 构建 NestJS API 镜像，启动时自动 `prisma migrate deploy` |
| `docker/nginx.Dockerfile` | 构建用户端/管理端两个前端，并打进 Nginx 镜像 |
| `docker/nginx.conf` | 反向代理：`/`→用户端、`/cms/`→管理端、`/api`+`/uploads`+`/video`→API（默认 HTTP，含 HTTPS 注释块） |
| `docker-compose.yml` | 编排 postgres + api + nginx |
| `.env.production.example` | 生产环境变量模板 |
| `scripts/deploy.sh` | 一键发布（git pull + 重建 + 健康检查） |

> **存储说明**：当前上传走本机磁盘（`local-storage.service.ts`），文件存在 ECS 的 `/opt/oisee/data/uploads`（compose 已挂卷持久化），由 API 经 Nginx 回源。你购买的 OSS 暂未接入；待接入后把存储层换成 `ali-oss` 即可，部署编排不变。

## 1. ECS 初始化（一次性）

```bash
# 系统更新 + Docker + 基础工具
apt update && apt upgrade -y
curl -fsSL https://get.docker.com | bash
apt install -y git ufw fail2ban

# 防火墙（与阿里云安全组冗余）
ufw allow 22/tcp && ufw allow 80/tcp && ufw allow 443/tcp && ufw --force enable
systemctl enable --now fail2ban
```

阿里云**安全组**放行：22（限你的 IP）、80、443。

## 2. 拉代码 + 建目录

```bash
mkdir -p /opt/oisee/{data/{postgres,uploads,video,nginx/ssl,logs/nginx},backups}
cd /opt/oisee
git clone https://github.com/BruceJong/OISEE.git repo
cd repo
```

## 3. 配置环境变量

```bash
cp .env.production.example /opt/oisee/.env.production
chmod 600 /opt/oisee/.env.production
nano /opt/oisee/.env.production          # 替换所有 __CHANGE_ME__
#   - 数据库密码（OISEE_DB_PASSWORD 与 OISEE_DATABASE_URL 里的密码必须一致）
#   - JWT/主密钥：openssl rand -hex 32
#   - 初期用 IP 测试：OISEE_WEB_ORIGIN / OISEE_ADMIN_ORIGIN / *_BASE_URL 填 http://<ECS公网IP>
```

## 4. 放入演示视频（可选）

`data/video/` 是运行时目录，不在 git 里。仓库带了一份样例：

```bash
cp /opt/oisee/repo/docs/prototype/sample_video.mp4 /opt/oisee/data/video/
```

## 5. 启动

```bash
cd /opt/oisee/repo
docker compose --env-file /opt/oisee/.env.production up -d --build
# 首次构建较久（装依赖+构建两个前端+API）。2C4G 若内存吃紧，见“常见问题”。
```

api 容器启动时会自动执行 `prisma migrate deploy`（应用全部 8 个迁移）。确认：

```bash
docker compose logs api | grep -i migrat
docker compose ps        # 三个容器都应 healthy / running
```

## 6. 灌入超管账号（仅首次）

```bash
docker compose exec api pnpm exec tsx prisma/seed.ts
```

> ⚠️ `prisma/seed.ts` 同时会灌入一批**示例内容**，其图片 URL 硬编码成 `http://localhost:3000/...`（开发期遗留）。生产环境这些示例图会裂；正式内容你都会在 CMS 里重新维护，所以无碍。若只想要超管不想要示例内容，告诉我，我帮你拆一个 system-only seed。

## 7. 验证

- 浏览器开 `http://<ECS公网IP>/` → 用户端加载
- `http://<ECS公网IP>/cms/` → 管理端登录页（用第 6 步的超管账号登录）
- `curl http://<ECS公网IP>/health` → `{"code":0,"data":{"status":"ok",...}}`

## 8. 域名 / 备案 / HTTPS（上线对外前）

1. 面向青少年的科普站需**国内 ICP 备案**，先去阿里云提交（耗时最长，尽早办）。
2. 备案下来后：域名 A 记录 → ECS 公网 IP。
3. 申请阿里云免费 DV 证书，下载后放 `/opt/oisee/data/nginx/ssl/`。
4. 编辑 `docker/nginx.conf`：启用底部 HTTPS 注释块、把 80 改成跳转 443、填你的域名。
5. 把 `.env.production` 里的 `*_ORIGIN` / `*_BASE_URL` 改成 `https://你的域名`。
6. `bash scripts/deploy.sh` 重新发布。

## 9. 日常发布

本地 `git push origin main` 后，在服务器：

```bash
bash /opt/oisee/repo/scripts/deploy.sh
```

## 10. 备份（建议）

```bash
# 手动备份一次
docker compose exec -T postgres pg_dump -U oisee oisee | gzip > /opt/oisee/backups/oisee-$(date +%Y%m%d-%H%M%S).sql.gz
# 每日 02:00 自动（crontab -e）
0 2 * * * docker compose -f /opt/oisee/repo/docker-compose.yml exec -T postgres pg_dump -U oisee oisee | gzip > /opt/oisee/backups/oisee-$(date +\%Y\%m\%d).sql.gz
```

媒体目录 `data/uploads` 也建议纳入备份（本机磁盘存储期间）；接入 OSS 后改由 OSS 冗余托管。

## 常见问题

- **2C4G 构建 OOM**：Vite 构建吃内存。临时加 swap：
  `fallocate -l 2G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile`
- **前端 `tsc -b` 报类型错导致 build 失败**：本地先 `pnpm -r typecheck` 修干净再推。
- **改了代码但容器没更新**：`deploy.sh` 用了 `--build`；必要时 `docker compose build --no-cache <service>`。
- **看日志**：`docker compose logs --tail=200 -f api`
